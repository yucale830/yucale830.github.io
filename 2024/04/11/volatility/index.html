VOLATILITY：内存取证工具

简介：
Volatility 是一个开源的内存取证框架，主要用于分析和调查计算机内存中的数据
它可以帮助取证人员、安全研究人员和调查人员在进行数字取证、恶意代码分析和安全事件响应时，从内存中提取关键信息
Volatility 提供了各种插件，可以用于识别进程、网络连接、文件系统状态、注册表键值等，以帮助用户分析和理解系统状态、行为和事件
通过分析内存数据，Volatility 可以揭示计算机系统的活动历史、运行的进程、打开的文件、网络连接以及其他重要信息，从而为调查和取证提供有力支持

输入-h指令 中文对照↓↓
F:\dzqz\tool\volatility3>python vol.py -h
Volatility 3 Framework 2.5.2
用法：volatility [-h] [-c CONFIG]
[--parallelism [{processes,threads,off}]]
[-e EXTEND] [-p PLUGIN_DIRS] [-s SYMBOL_DIRS]
[-v] [-l LOG] [-o OUTPUT_DIR] [-q] [-r RENDERER]
[-f FILE] [--write-config]
[--save-config SAVE_CONFIG] [--clear-cache]
[--cache-path CACHE_PATH] [--offline]
[--single-location SINGLE_LOCATION]
[--stackers [STACKERS ...]]
[--single-swap-locations [SINGLE_SWAP_LOCATIONS ...]]
plugin ...

一个开源的内存取证框架

选项：
-h, --help 显示此帮助消息并退出，要查看特定插件选项，请使用 'volatility <pluginname> --help'
-c CONFIG, --config CONFIG
从 json 文件加载配置
--parallelism [{processes,threads,off}]
启用并行处理（如果未指定参数，则默认关闭）
-e EXTEND, --extend EXTEND
使用新的（或更改的）设置扩展配置
-p PLUGIN_DIRS, --plugin-dirs PLUGIN_DIRS
以分号分隔的路径列表，用于查找插件
-s SYMBOL_DIRS, --symbol-dirs SYMBOL_DIRS
以分号分隔的路径列表，用于查找符号
-v, --verbosity 增加输出详细程度
-l LOG, --log LOG 将日志输出到文件和控制台
-o OUTPUT_DIR, --output-dir OUTPUT_DIR
生成文件的输出目录
-q, --quiet 移除进度反馈
-r RENDERER, --renderer RENDERER
确定如何呈现输出（快速、无、csv、pretty、json、jsonl）
-f FILE, --file FILE 如果未定义 single-location，则是 single-location=file:// 的简写
--write-config 将配置 JSON 文件写出到 config.json
--save-config SAVE_CONFIG
将配置 JSON 文件保存到文件
--clear-cache 清除所有短期缓存项
--cache-path CACHE_PATH
更改用于存储缓存的默认路径
--offline 不在线搜索额外的 JSON 文件
--single-location SINGLE_LOCATION
指定一个基本位置进行堆叠
--stackers [STACKERS ...]
堆叠器列表
--single-swap-locations [SINGLE_SWAP_LOCATIONS ...]
指定一组用于 single-location 的交换层 URI
